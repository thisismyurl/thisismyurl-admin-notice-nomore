# Contributing

Pull requests welcome.

## Standards

The plugin follows the [WordPress Coding Standards](https://developer.wordpress.org/coding-standards/). Keep output escaped, inputs sanitized, capability checks in place, and direct file access guarded.

## Workflow

1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/your-change`).
3. Commit your changes with a clear message — explain *why*, not just what.
4. Push to your fork and open a Pull Request against `main`.

For larger changes, open an issue first to discuss scope.

## Scope

This plugin is intentionally tiny: it suppresses admin notices and does nothing else. Bug reports and security fixes are always welcome. New features are weighed against that single purpose — single-purpose plugins age better than feature-creep ones.
